# 股商测试 MBTI — 分析逻辑全解

---

## 一、理论框架

本测试的整体架构借用了 MBTI 的 **四维度 × 二进制倾向 = 16 型人格** 的骨架，但四个维度的定义完全重构为与 A 股投资行为直接对应的心理测量维度。

### MBTI 与本测试的维度对照

| MBTI 原始维度 | 股商测试维度 | 左极（高分倾向） | 右极（低分倾向） |
|---|---|---|---|
| E/I 外向/内向 | D1 决策来源 | 独立分析 (I) | 跟风从众 (H) |
| S/N 感觉/直觉 | D2 信息偏好 | 技术博弈 (T) | 价值发现 (V) |
| T/F 思考/情感 | D3 执行模式 | 纪律至上 (D) | 情绪驱动 (E) |
| J/P 判断/感知 | D4 时间偏好 | 短线猎手 (S) | 长期主义 (L) |

**设计原则**：每个维度锚定一个在行为金融学中有明确依据的投资行为轴：

- **D1** 对应「羊群效应」(Herd Behavior)
- **D2** 对应「信息处理偏好」(Technical vs Fundamental Analysis)
- **D3** 对应「处置效应」与「情绪调节能力」(Disposition Effect & Emotional Regulation)
- **D4** 对应「时间偏好」(Time Preference / Intertemporal Choice)

## 二、题库结构

### 2.1 题目规格

- **题目总数**：24 题
- **每维度**：6 题
- **量表类型**：5 点李克特量表
  - 1 = 非常不同意
  - 2 = 比较不同意
  - 3 = 一般/不确定
  - 4 = 比较同意
  - 5 = 非常同意
- **未作答默认值**：3（中性）

### 2.2 各维度题目分布

#### D1：独立分析 ↔ 跟风从众

**低分(<50%)→跟风(H)**：容易受群体情绪、大 V 和社交信息左右

| Q10 | forward | 关注主力资金流向 vs 公司盈利能力 |

| Q12 | reverse | 因商业模式优秀而买入（不在乎图形难看）|

**高分(≥50%)→技术(T)**：以 K 线、量价、指标为核心决策依据
**低分(<50%)→价值(V)**：以财报、商业模式、基本面估值为核心决策依据

#### D3：纪律至上 ↔ 情绪驱动

| 题号 | 方向 | 测量内容 |
|---|---|---|
| Q13 | forward | 止损执行的果断程度（肌肉记忆）|

| Q15 | forward | 是否有书面交易计划（非凭感觉）|
| Q16 | reverse | 处置效应：赚 15% 就想止盈，亏 30% 死扛 |
| Q17 | reverse | 报复交易倾向（亏了加大仓位干回来）|
| Q18 | forward | 连续止损后的情绪稳定度（概率思维）|

**高分(≥50%)→纪律(D)**：机械执行、概率思维、系统化交易
**低分(<50%)→情绪(E)**：被贪婪恐惧驱动、报复交易、止盈过早亏损死扛

#### D4：短线猎手 ↔ 长期主义

| 题号 | 方向 | 测量内容 |
|---|---|---|
| Q19 | forward | 每日看盘频率（>10 次 = 交易成瘾倾向）|

| Q22 | reverse | 定投+资产配置 vs 龙头战法+打板 |
| Q23 | forward | 3 天无操作即焦虑（资金闲置焦虑）|
| Q24 | reverse | 20% 波动不动摇，关注 3 年后发展 |

**高分(≥50%)→短线(S)**：高频交易、追求即时反馈、资金周转焦虑
**低分(<50%)→长期(L)**：复利信仰、无视短期波动、终局思维

## 三、评分算法（核心引擎）

评分引擎位于 `src/utils/scoring.js`，入口函数为 `calculateScores(answers)`。

### 3.1 输入

```js
answers = {
  1: 4,   // 用户对 Q1 的选择: 4 = 比较同意
  2: 2,   // 用户对 Q2 的选择: 2 = 比较不同意
  // ... 24 个题目的得分（1-5）
}
```

### 3.2 第一步：反向计分

```js
effectiveScore = q.reverse ? (6 - rawScore) : rawScore
```

**为什么需要反向计分？**

题库中有些题目「高分 = 倾向右侧」，有些「高分 = 倾向左侧」。反向计分将所有题目统一为 **高分 = 倾向右侧** 的方向。

以 D1 为例：右侧 = 跟风从众(H)。

- Q1 是 reverse 题，rawScore=5（非常同意「群友喊冲我就冲」），effectiveScore = 6-5 = **1**，说明用户**不倾向右侧（不跟风=倾向独立）**
- Q2 是 forward 题，rawScore=5（非常同意「坚持自己判断」），effectiveScore = **5**，说明用户**不倾向右侧（坚持自己=倾向独立）**

经过这步处理后，所有维度的 effectiveScore **越高 = 越倾向该维度的右侧**。

### 3.3 第二步：按维度汇总

```js
dimensions['D1'] = Σ(effectiveScore of Q1~Q6)
dimensions['D2'] = Σ(effectiveScore of Q7~Q12)
dimensions['D3'] = Σ(effectiveScore of Q13~Q18)
dimensions['D4'] = Σ(effectiveScore of Q19~Q24)
```

每维度 6 题，得分范围：**6 ~ 30**

### 3.4 第三步：归一化到百分比

```js
percentage = Math.round(((raw - 6) / (30 - 6)) * 100)
```

分子 `(raw - 6)` 将范围平移为 0~24
分母 `(30 - 6) = 24` 是最大跨度
结果：0% ~ 100%

| 原始分 | 百分比 | 含义 |
|---|---|---|
| 6 | 0% | 完全倾向左侧（极度独立/技术/纪律/短线）|
| 18 | 50% | 恰好位于分界线 |
| 30 | 100% | 完全倾向右侧（极度跟风/价值/情绪/长期）|

### 3.5 第四步：二分判定

```js
if (pct >= 50) → 倾向右侧
else           → 倾向左侧
```

这里的 **二分阈值是 50%**（对应原始分 18），含义是：

- 百分比 ≥ 50%：用户在此维度上**更偏向右侧标签**
- 百分比 < 50%：用户在此维度上**更偏向左侧标签**

### 3.6 第五步：生成人格类型码

```js
tendencies = {
  D1: 'I' | 'H',   // I=独立(左)  H=跟风(右)
  D2: 'T' | 'V',   // T=技术(左)  V=价值(右)
  D3: 'D' | 'E',   // D=纪律(左)  E=情绪(右)
  D4: 'S' | 'L',   // S=短线(左)  L=长期(右)
}

typeCode = D1倾向 + D2倾向 + D3倾向 + D4倾向
// 例如: "ITDS" 或 "HVEL"
```

4 个维度，每维度 2 种倾向，共 **2⁴ = 16 种**组合。

## 五、雷达图可视化

雷达图数据来自 `getRadarData(percentages)` 函数。

### 5.1 转换逻辑

```js
function getRadarData(percentages) {
  return [
    { dimension: '独立分析', value: 100 - percentages.D1 },  // 翻转：高分=更独立
    { dimension: '技术博弈', value: percentages.D2 },         // 正向：高分=更技术
    { dimension: '纪律至上', value: 100 - percentages.D3 },  // 翻转：高分=更纪律
    { dimension: '长期主义', value: percentages.D4 },         // 正向：高分=更长期
  ]
}
```

四个维度在雷达图上统一呈现为「正面能力」的 0-100 分数：

| 雷达轴 | 高分含义 | 转换方式 |
|---|---|---|
| 独立分析 | 越独立越高 | `100 - D1%`（D1%越高=越跟风，所以翻转）|
| 技术博弈 | 越技术越高 | 直接用 D2%（D2%越高=越技术）|
| 纪律至上 | 越纪律越高 | `100 - D3%`（D3%越高=越情绪，所以翻转）|
| 长期主义 | 越长期越高 | 直接用 D4%（D4%越高=越长期）|

雷达图读法：**面积越大的方向，代表你在这方面的正向能力越强。**

## 六、端到端评分示例

假设一位用户小明，他的答题情况如下：

### 6.1 D1 维度答题与计分

| 题号 | 题目简述 | reverse | rawScore | effectiveScore |
|---|---|---|---|---|
| Q1 | 群友喊冲就跟 | true | 2 | 6-2=**4**（低跟风=独立）|
| Q2 | 大 V 反对仍坚持自己 | false | 4 | **4**（坚持=独立）|
| Q3 | 全网唱空怀疑自己 | true | 3 | 6-3=**3** |
| Q4 | 朋友圈晒收益→踏空焦虑 | true | 4 | 6-4=**2** |
| Q5 | 自己读公告而非刷论坛 | false | 3 | **3** |
| Q6 | 随大流清仓后大腿拍断 | true | 2 | 6-2=**4** |

D1 原始总分 = 4+4+3+2+3+4 = **20**

D1 百分比 = (20-6)/(30-6) × 100 = 14/24 × 100 = **58%**

58% ≥ 50% → D1 倾向 **H（跟风从众）**

### 6.2 全部维度得分

| 维度 | 原始分 | 百分比 | 二分判定 |
|---|---|---|---|
| D1 | 20 | 58% | H（跟风）|
| D2 | 10 | 17% | T（技术）|
| D3 | 22 | 67% | E（情绪）|
| D4 | 8 | 8% | S（短线）|

类型码 = **H** + **T** + **E** + **S** = **HTES**

### 6.3 结果解析

小明是 **HTES「韭菜特种兵」🥬**：

- 跟风操作（H）+ 懂技术但情绪化使用（T+E）+ 超短线（S）
- 雷达图呈现：独立分析 42%、技术博弈 17%、纪律至上 33%、长期主义 8%
- 这是最需要成长的类型，进阶路线：HTES → HTDS → ITDS

## 七、关键设计决策

### 7.1 为什么用 50% 作为二分阈值？

百分比 50% 对应原始分 18 分。在 6 题 × 5 点量表中，每题 neutral 答案（3 分）恰好合计 18 分。将阈值设为 50% 意味着：**恰好全选「一般」的用户会被划分到右侧**。

这是有意为之——在行为金融学中，「不确定」往往意味着该维度的弱势。例如在 D3 维度，「一般/不确定」自己的纪律性 = 实际上纪律性不够强。

### 7.2 为什么每题只用 5 点而非 7 点？

- 5 点降低认知负担，提高完答率
- 移动端窄屏下 5 个选项刚好一行展示
- 每题有效区分 5 个等级，6 题共 30 个等级，足够细粒度

### 7.3 为什么每维度 6 题而非更多？

- 24 题总时长约 3-5 分钟，是社交媒体传播的黄金时长
- 6 题足以覆盖一个维度的不同侧面
- 每题独立贡献，各维度内 Cronbach's α 理论上可达 0.7+

## 八、代码调用链路

```
用户点击「开始测试」
  → WelcomePage → onStart()
    → App.setState(page='quiz')

用户逐题作答（24题）
  → QuizPage 管理 current/answers/selectedScore 状态
  → 每题选择后 setAnswers({...answers, [qid]: score})
  → 最后一题点击「查看结果」
    → onComplete(finalAnswers)
      → calculateScores(finalAnswers)
        → 反向计分 → 维度汇总 → 归一化 → 二分判定 → 生成 typeCode
      → App.setState(page='result', result=scores)

结果页渲染
  → ResultPage 接收 result={typeCode, percentages, tendencies, rawScores}
    → personalities[typeCode] 查找人格数据
    → getRadarData(percentages) 生成雷达图数据
    → 依次渲染 10 个内容区段
```

## 九、总结

整个分析引擎的逻辑链条：

```
24道李克特题目
  → 反向计分统一方向
    → 4维度独立汇总（每维度6-30分）
      → 归一化百分比（0%-100%）
        → 50%阈值二分判定
          → 拼接4字母类型码（如 ITDS）
            → 查表获取16型人格全量数据
              → 雷达图可视化 + 10区段结果展示
```

这个架构的优点是 **解释性强、可审计**——每一步都可以反推，用户问「为什么我是这个类型」时，可以精确追踪到每道题的计分。同时，题库和人格体系与评分引擎完全解耦，后续可以独立迭代题库而不改引擎。
